# 第五章 指派问题：匈牙利方法例题

原始耗时矩阵：

```text
        英文版 日文版 德文版 俄文版
甲        2     15    13     4
乙       10      4    14    15
丙        9     14    16    13
丁        7      8    11     9
```

目标：每位译员分配一个版本，每个版本只分配给一位译员，使总耗时最小。

## 1. 行约简

每一行减去本行最小元素：

```text
甲行最小值 2
乙行最小值 4
丙行最小值 9
丁行最小值 7
```

得到：

```text
0  13  11  2
6   0  10 11
0   5   7  4
0   1   4  2
```

## 2. 列约简

各列最小值为：

```text
第1列 0
第2列 0
第3列 4
第4列 2
```

每一列再减去本列最小元素，得到约化费用矩阵：

```text
0  13   7  0
6   0   6  9
0   5   3  2
0   1   0  0
```

## 3. 寻找独立零元素

零元素位置：

```text
甲：英文版、俄文版
乙：日文版
丙：英文版
丁：英文版、德文版、俄文版
```

按圈零元素法：

1. 乙行只有一个零元素，圈 `(乙, 日文版)`。
2. 丙行只有一个零元素，圈 `(丙, 英文版)`。
3. 英文版列已被占用，丁行剩下可选零元素为德文版、俄文版。
4. 甲行英文版列已被占用，只能圈 `(甲, 俄文版)`。
5. 丁行只能圈 `(丁, 德文版)`。

得到4个独立零元素：

```text
甲 -> 俄文版
乙 -> 日文版
丙 -> 英文版
丁 -> 德文版
```

因为矩阵阶数为4，独立零元素个数也为4，所以已经得到最优指派方案。

## 4. 最小总耗时

回到原始耗时矩阵取数：

```text
甲 -> 俄文版：4
乙 -> 日文版：4
丙 -> 英文版：9
丁 -> 德文版：11
```

总耗时：

```text
4+4+9+11=28
```

最优方案：

```text
甲译俄文版，乙译日文版，丙译英文版，丁译德文版
最小总耗时 = 28
```

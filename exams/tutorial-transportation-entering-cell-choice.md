# 运输问题：能不能换一个空格进基

## 问题

用户问：

```text
是不是也可以保留 x12、x32、x33 做回路，在 x13 添加？
```

## 1. 结论

可以形成闭回路，但不应该选 `x13` 进基。

原因：

```text
闭回路能不能形成是一件事；
该不该让这个空格进基是另一件事。
```

运输问题中，先由检验数决定“哪个空格可以进基”，再为这个进基格找闭回路。

## 2. 对 x13 的闭回路

若强行让 `x13` 进基，可以用：

```text
(1,3)+ -> (1,2)- -> (3,2)+ -> (3,3)- -> 回到 (1,3)
```

这个闭回路形式上是成立的。

## 3. 但是 x13 的检验数不允许它进基

初始位势为：

```text
u1 = 0, u2 = -5, u3 = -5
v1 = 6, v2 = 9, v3 = 7, v4 = 7
```

课件检验数：

```text
r_ij = u_i + v_j - c_ij
```

对 `x13`：

```text
r13 = u1 + v3 - c13
    = 0 + 7 - 10
    = -3
```

这是最小化运输问题，课件规则为：

```text
所有空格 r_ij <= 0，则最优；
若有 r_ij > 0，则可改进；
一般选最大正检验数进基。
```

因为：

```text
r13 = -3 < 0
```

所以 `x13` 不是改进方向，不能选它进基。

## 4. 强行调整会怎样

若沿回路：

```text
(1,3)+ -> (1,2)- -> (3,2)+ -> (3,3)-
```

负号格子为：

```text
x12 = 6
x33 = 6
```

所以：

```text
θ = 6
```

费用变化为：

```text
(加号成本 - 减号成本) * θ
= (c13 + c32 - c12 - c33) * 6
= (10 + 4 - 9 - 2) * 6
= 3 * 6
= 18
```

费用会从：

```text
102
```

变成：

```text
120
```

费用上升，所以不能这样改。

## 5. 正确规则

```text
第一步：先算所有空格检验数；
第二步：只从 r_ij > 0 的空格中选进基格；
第三步：通常选最大正检验数；
第四步：为这个选定的进基格找闭回路。
```

## 6. 本题第一次应选

初始检验数中：

```text
r11 = 4 > 0
r22 = 1 > 0
```

最大正检验数是：

```text
r11 = 4
```

所以应选：

```text
x11 进基
```

而不是 `x13`。
